Chapter divider pages (`CHAPTER – N` / `CHAPTER NAME`, twistedLines1 border, 21×27.8cm) are generated by [modules/chapter_titles.py](modules/chapter_titles.py) and inserted by `PDFMerger.insert_chapter_title_pages`.

**Why the ordering matters:** dividers are inserted AFTER the header/footer step (main.py Step 6), on purpose. The user wanted dividers to have NO class header and NO page-number footer, and to be unnumbered so they don't shift the content footer/index page numbers. Inserting them post-numbering achieves all three at once.

**How to apply:** if you change pipeline step order, keep divider insertion strictly after `add_header_footer` and before the index combine. Chapter detection (`_build_chapter_list`) uses the same "chapter_name changed" rule as [[spectropy_index]] so divider numbers match the index. Conversion uses docx2pdf (Word COM) with `pythoncom.CoInitialize()` for the GUI daemon thread — not LibreOffice.

Confirmed working end-to-end by the user 2026-06-02.
